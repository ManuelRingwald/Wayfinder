# WF2-12.1 — Tenant-Context-Middleware + Authenticator-Factory

> **Stufe:** 1 (Umsetzung) · **Paket:** WF2-12, Häppchen 12.1 · **Einstufung:**
> 🔒 S4 · Opus 4.8 · **Grundlage:** ADR 0005/0006, baut auf WF2-10 (`pkg/store`)
> und WF2-11 (`pkg/auth`).

## Warum (fachlich)

Hier kommen **Identität (pkg/auth)** und **Persistenz (pkg/store)** zusammen: jede
Anfrage wird zu einem **Mandanten** aufgelöst und **fail-closed** abgewiesen, wenn
kein gültiger Mandant feststeht. Das ist der **Enforcement-Punkt der
Cross-Tenant-Isolation am HTTP-Rand** (NFR-SEC-003/004) und die Vorstufe zum
sicherheitskritischen Kern (Stufe 2: mandanten-isolierter Stream).

## Was (technisch)

- **`pkg/auth/factory.go`** — `Config` (aus `WAYFINDER_AUTH_*`) + `NewAuthenticator(ctx, cfg)`:
  baut den Authenticator je `Mode`. **Fail-closed-Konfiguration:** fehlende
  Pflichtangaben sind ein Fehler (builtin ohne Session-Key, proxy ohne Issuer/
  Audience) — ein halb-konfigurierter sicherer Modus darf nicht still degradieren.
- **`pkg/tenant`** (neues Paket, Brücke auth↔store):
  - `Identity` (TenantID, UserID, Subject, Role) — der Isolations-Anker; jede
    spätere scoped Query/Stream (WF2-21) ist auf `TenantID` gekeyt.
  - Context-Helfer `WithIdentity`/`FromContext`.
  - `UserLookup`-Interface (`*store.UserRepo` erfüllt es; Tests nutzen ein Fake).
  - **`Middleware(authenticator, users, logger)`** — authentifiziert (→ Subject),
    löst **subject→user→tenant** über `UserRepo.GetBySubject` auf, legt die
    `Identity` in den Request-Kontext und ruft `next`. **Fail-closed:** ohne
    gültige Identität *oder* bei unbekanntem Subject → **401**, `next` wird **nie**
    erreicht; die Ursache wird dem Client nicht geleakt, unerwartete Lookup-Fehler
    (z. B. DB down) werden geloggt.

## Tests (DB-frei)

- `pkg/auth/factory_test.go`: korrekter Authenticator je Modus; **Fail-closed-
  Validierung** (builtin ohne Key, proxy ohne Issuer/Audience, unbekannter Modus
  → Fehler).
- `pkg/tenant/tenant_test.go`: Context-Round-Trip; **Erfolgspfad** (Identity im
  Kontext); **Fail-closed** in drei Fällen (Auth schlägt fehl, unbekanntes
  Subject, Lookup-/DB-Fehler) → 401, `next` nie aufgerufen. Fake-Authenticator +
  Fake-`UserLookup` (die reale `GetBySubject` ist bereits in WF2-10.2 gegen
  PostgreSQL verifiziert).

Gates grün: `go build/vet/test`, `gofmt`.
