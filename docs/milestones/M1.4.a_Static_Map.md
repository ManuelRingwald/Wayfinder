# M1.4.a — Statische Karte anzeigen

## Fachlich

Eine ASD (Air Situation Display) zeigt dem Lotsen die Luftlage auf einem
geografischen Hintergrund — der Karte. Bevor Live-Tracks dargestellt werden
können (M1.4.b/c), braucht es diesen Hintergrund: eine im Browser gerenderte
2D-Karte mit konfigurierbarem Ausschnitt (Mittelpunkt, Zoom) und
Kartenkachel-Quelle.

## Technisch

- **`internal/webui`**: neues Go-Package, das die Frontend-Assets
  (`index.html`, `app.js`) per `//go:embed` ins Binary einbettet und über
  `http.FileServer` ausliefert. Dadurch bleibt Wayfinder ein einzelnes,
  deploybares Binary (kein separater Frontend-Build/-Deploy nötig).
- **MapLibre GL JS** (siehe Glossar) wird per CDN eingebunden und rendert die
  Karte in `#map`.
- **Konfiguration**: `cmd/wayfinder/main.go` liest
  `WAYFINDER_MAP_CENTER_LAT`, `WAYFINDER_MAP_CENTER_LON`, `WAYFINDER_MAP_ZOOM`
  und `WAYFINDER_MAP_STYLE_URL` aus der Umgebung (Default: Frankfurt am Main,
  Zoom 8, OpenStreetMap-Raster-Kacheln ohne API-Key). Der Endpunkt
  `/api/map-config` liefert diese Werte als JSON; `app.js` lädt sie beim
  Start und initialisiert die Karte entsprechend.
- **Routen** auf dem WebSocket-Server (Port 8081): `/` (Frontend), `/ws`
  (bestehend), `/api/map-config` (neu).

## Architektur-Hinweis

Die Konfigurierbarkeit von Kartenmittelpunkt/-zoom/-kachel-Quelle über
Umgebungsvariablen ist Teil des Cloud-native-Grundsatzes (CLAUDE.md §7) —
ein Wayfinder-Deployment für einen anderen Flughafen/Sektor braucht keine
Code-Änderung, nur andere Env-Werte.

## Tests

- `cmd/wayfinder/main_test.go`: `/api/map-config` liefert Default-Style
  (eingebettetes OSM-Raster) bzw. eine konfigurierte Style-URL korrekt als
  JSON.
- Manuell verifiziert: `/`, `/app.js` und `/api/map-config` liefern HTTP 200
  bei laufendem Server.
