# ASD-014.4 — Namens-Picker für den AoR-Editor

> **Kontext:** Abrundung von **ASD-014** (ADR 0021). Löst die in ASD-014.3
> vermerkte „ehrliche Grenze" (ID-Eingabe) auf: der Admin wählt die
> Verantwortungsbereich-Lufträume **nach Namen** statt rohe IDs zu tippen.
> **Kein CAT062-Bezug, keine neuen Env-Variablen.**

## Fachlich — warum

Rohe OpenAIP-IDs sind für den Betreiber nicht auffindbar. Mit einem Namens-Picker
(„HAMBURG CTR") wird das AoR-Setzen bedienbar; gespeichert wird weiterhin die
**stabile `id`** (namens-driftfest über AIRAC-Zyklen).

## Technisch — wie

### Backend — neuer Admin-Endpunkt
`GET /api/admin/tenants/{id}/airspaces` (hinter `requireAdmin`) liefert die
Luftraum-Liste **des Mandanten `{id}`** aus dem **bereits vorhandenen** Aeronautik-
Cache — **kein neuer Fetch, keine neue Datenquelle**.
- **`pkg/adminapi/adminapi_airspaces.go`:** `AirspaceLister`-Interface +
  `AirspaceOption` (`{id, name, type?, icao_class?}`), `WithAirspaceLister`,
  Handler `getTenantAirspaces` (nil-Lister → 404, `tenantExists`-Check,
  `requireAdmin`). Route in `adminapi.go`.
- **`cmd/wayfinder/aero.go`:** `aeroAirspaceLister` adaptiert
  `aeronautical.Registry.Serve({id}, KindAirspace)`; `projectAirspaces`
  (reine, testbare Funktion) verwirft id-lose Features, liest die numerischen
  Properties **robust** (int bei frischem Fetch, float64 nach Cache-Hydration),
  sortiert nach Name. Verdrahtet über `WithAirspaceLister` in `main.go`. Damit
  bleibt `pkg/adminapi` transport-agnostisch (kein Import der GeoJSON-Typen).

### Frontend — Picker
- **`stores/admin.js`:** `loadTenantAirspaces(tenantId)`.
- **`AdminTenantDetail.vue`:** das AoR-Feld ist jetzt ein **`v-autocomplete`**
  (multiple, chips) mit `item-title="name"`, `item-value="id"`; die Items kommen
  aus dem neuen Endpunkt (`loadAirspaces()` in `onMounted`). Das Modell bleibt
  `form.aorAirspaceIds` (IDs). **`airspaceItems`** (computed) mischt gewählte,
  aber **nicht (mehr) gecachte** IDs als synthetische Einträge dazu — so geht eine
  gespeicherte AoR-ID nie still verloren. Leerer Cache → `no-data`-Hinweis „erst
  OpenAIP konfigurieren".

## Ehrliche Grenze

Die Liste zeigt, **was für den Mandanten gecacht ist**. Hat der Mandant noch kein
OpenAIP konfiguriert/geholt, ist sie leer (Hinweis im Feld). Der Server validiert
die Auswahl weiterhin autoritativ (`validateView`).

## Schnittstellen-Wirkung

**Keine** am CAT062. Neuer interner Admin-Endpunkt, additiv; `pkg/aeronautical`
unverändert (nur lesend über `Serve` genutzt).
