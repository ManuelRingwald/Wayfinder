# WF2-10.3b — Repository: View-Configs (Persistenz-Schicht komplett)

> **Stufe:** 1 (Umsetzung) · **Paket:** WF2-10, Häppchen 10.3b (Abschluss von
> WF2-10) · **Einstufung:** S3 · Sonnet 4.6 (+Opus-Review) · **Grundlage:** ADR
> 0006, baut auf WF2-10.1/10.2/10.3a.

## Warum (fachlich)

`view_configs` ist die **mandanten-/nutzer-spezifische Sicht** (Zentrum, Zoom,
Area-of-Interest, FL-Band, Layer-Schalter). Sie wird beim Session-Aufbau (WF2-30)
gelesen, um dem Nutzer die richtige Karte zu liefern, und im Admin-Bereich
(WF2-31/32) gesetzt. Modell: ein **Tenant-Default** (eine Zeile mit `user_id
IS NULL`) plus optionale **Nutzer-Übersteuerung** — `GetEffective` liefert die
Übersteuerung, sonst den Default. Damit ist die Persistenz-Schicht **WF2-10
vollständig** (alle sechs Tabellen haben Repositories).

## Was (technisch)

- **Migration `00002_view_config_user_unique.sql`** — wandelt den Nutzer-Index in
  einen **Partial-Unique-Index** (`WHERE user_id IS NOT NULL`), sodass es **eine
  Übersteuerung je Nutzer** gibt und `UpsertUserOverride` `ON CONFLICT (user_id)`
  nutzen kann. Zeigt zugleich die **Schema-Evolution** des In-House-Runners
  (mehrere Migrationen in aufsteigender Reihenfolge).
- `view_configs.go`:
  - Typen `BBox` (AOI) und `ViewConfig` (`UserID *int64`: nil = Tenant-Default;
    `AOI *BBox`, `FLMin`/`FLMax *int` optional; `Layers map[string]bool`).
  - `UpsertTenantDefault` (`ON CONFLICT (tenant_id) WHERE user_id IS NULL`),
    `UpsertUserOverride` (`ON CONFLICT (user_id) WHERE user_id IS NOT NULL`).
  - `GetTenantDefault`, `GetUserOverride`, **`GetEffective`** (Übersteuerung →
    Fallback Default → `ErrNotFound`).
  - JSONB `aoi`/`layers` über `$n::jsonb` + `toJSONB`/`fromJSONB`; `aoi` nullable
    (nil → SQL NULL), `layers` immer JSON-Objekt (`{}` wenn leer).

## Tests & Verifikation

- **DB-frei:** `TestViewJSONParams` (aoi nil → NULL-Param, layers leer → `{}`).
- **Integration (`view_configs_test.go`):** Default-Upsert + Round-Trip
  (AOI/FL/Layers), **In-Place-Update** (zweiter Upsert = gleiche Zeile, Felder
  ersetzt, AOI/FL geleert), Nutzer-Übersteuerung idempotent, `GetEffective`
  (Default ohne Override, dann Override). `TestLoadMigrations` auf **zwei**
  Migrationen erweitert.
- **Real gegen PostgreSQL 16** via `scripts/pg-test.sh` (wendet beide Migrationen
  an) — alle grün; Standard-`go test ./...` grün ohne DB.

Gates grün: `go build/vet/test`, `gofmt`, `scripts/pg-test.sh`.
