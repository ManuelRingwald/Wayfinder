# WF2-12.2 — Multi-Tenancy-Verdrahtung im Server (`main.go`)

> **Stufe:** 1 (Umsetzung) · **Paket:** WF2-12, Häppchen 12.2 · **Einstufung:**
> 🔒 S4 · Opus 4.8 · **Grundlage:** ADR 0005/0006, baut auf WF2-10/11/12.1.

## Warum (fachlich)

Bringt die in WF2-12.1 gebaute Tenant-Context-Middleware in den **laufenden
Server**: Identität (pkg/auth) + Persistenz (pkg/store) werden beim Start
verdrahtet, und der **Live-Datenstrom `/ws`** wird mandanten-gegated.
**Rückwärtskompatibel:** ohne `WAYFINDER_DB_URL` läuft Wayfinder **wie bisher**
als Single-Tenant-ASD (ADR 0005 §7, degenerierter Fall).

## Was (technisch)

`cmd/wayfinder/main.go`:

- **`Config`** um Multi-Tenancy-Felder erweitert (`DBURL`, `AuthMode`,
  `NoneSubject`, `SessionKey`, `SessionCookie`, `OIDCIssuer`, `OIDCAudience`);
  `loadConfig` liest `WAYFINDER_DB_URL`/`_AUTH_MODE`/`_OIDC_*`/`_SESSION_*`/
  `_NONE_SUBJECT`; `Config.authConfig()` projiziert auf `auth.Config`.
- **`setupTenancy(ctx, cfg, logger)`**: wenn `DBURL` leer → `(nil, nil, nil)`
  (Single-Tenant, Warn-Log). Sonst: `store.Open` + `store.Migrate` (Schema beim
  Start) + `auth.NewAuthenticator` (aus der Config, fail-closed bei
  Fehlkonfiguration) + `tenant.Middleware`. Gibt Middleware + Pool zurück (Pool
  vom Aufrufer geschlossen).
- **Verdrahtung in `main()`** (30-s-Setup-Timeout): bei aktiver Tenancy wird
  `/ws` mit `tenantMW(...)` umschlossen (der scoped Fan-out selbst folgt WF2-21);
  der **Legacy-Einzeltoken** (`WAYFINDER_AUTH_TOKEN`) wird dann **übersprungen**
  (die Tenant-Middleware löst ihn ab). Im Single-Tenant-Fall bleibt alles wie bisher.

`scripts/pg-test.sh` läuft jetzt über `./...` (damit auch der cmd-Integrationstest
mit echtem Postgres mitläuft).

## Tests

`cmd/wayfinder/tenancy_test.go`:

- **DB-frei:** `TestLoadConfigTenancyEnvVars` (Parsing + Default `none`),
  `TestSetupTenancyDisabledWithoutDB` (ohne DB → kein Middleware/Pool).
- **Real gegen PostgreSQL 16** (`scripts/pg-test.sh`): `TestSetupTenancyEnabled`
  verdrahtet die volle Kette (DB öffnen + migrieren + none-Authenticator +
  Middleware) und prüft **fail-closed** (ohne „default"-Nutzer → `401`) sowie die
  **Auflösung** (mit Tenant + „default"-Nutzer → `200`, Identity trägt die
  korrekte `TenantID`).

Doku: INSTALLATION §7 + TECHNICAL §6 um die neuen (jetzt **aktiven**)
ENV-Variablen ergänzt; Register FR-CFG-001 + NFR-SEC-004. Gates grün
(`go build/vet/test`, `gofmt`, `scripts/pg-test.sh`); `go 1.25` unverändert.

## Abgrenzung / Nächstes

- Welche Routen geschützt sind: **`/ws`** (Live-Bild). `/`, `/api/map-config`,
  Aeronautik- und Coverage-Endpoints bleiben vorerst offen (Referenz-/Shell-Daten);
  ihre Tenant-Skopierung folgt mit View-Config/Admin (WF2-30 ff.).
- **WF2-12.3:** builtin-Login-Handler (`/api/login`: Passwort via
  `auth.VerifyPassword` prüfen → `auth.MintSession`-Cookie) + Credential-Speicher
  (Migration: Passwort-Hash). Der **proxy-Modus** ist bereits voll funktionsfähig.
- Danach WF2-13 (Admin-Bootstrap: ersten Tenant/Nutzer anlegen), dann **Stufe 2**
  (scoped Fan-out, WF2-20/21/22).
